# 使用說明

## 這是什麼
`index.html` 是**完全獨立**的單一檔案,不再需要 `support.js`、`styles.css`、`portfolio.dc.js`。
直接雙擊用瀏覽器打開就能動(首頁動畫、自訂游標、WORKS 分類篩選、左右滑動瀏覽、
作品詳細頁、中英文切換、ABOUT、CONTACT 全部都正常運作)。

## 補圖片
把你的作品照片放進 `assets/` 資料夾,檔名照 `assets/REQUIRED_IMAGES.txt` 裡列的清單對照(共 17 張)。
檔名對上就會自動顯示;對不上或還沒放,會顯示「圖片待補」的灰色提示框,不會讓版面跑掉。

## 改文字內容
所有文案都集中在 `index.html` 最上面 `<script>` 區塊裡的 `PROJECTS` 陣列(從第一個有
`const PROJECTS = [` 的地方開始)。每個作品一段,中文欄位是 `xxxZh`,英文欄位是 `xxxEn`,
直接改文字就好,不用懂程式。`ABOUT` 頁的自我介紹在 `renderAbout()` 函式裡的 `aboutP1` / `aboutP2`。

## 待你確認/補上的東西
- 目前 `PROJECTS` 裡的文案,是延用你原始檔案 `portfolio_dc.js` 裡已經寫好的內容(君邑建設、
  立信、利都、煙波飯店等 12 個專案),我沒有更動文字本身,只是把它轉成不需要引擎的格式。
  如果你想要全部換成新的文案,把新內容傳給我,我直接幫你套進對應欄位。
- CONTACT 頁的電話目前還是原始檔案裡的佔位數字 `+886 9XX XXX XXX`,記得改成你的真實號碼
  (在 `renderContact()` 函式裡搜尋這串文字)。
- IG 連結我已經接上 `instagram.com/lai.jiachin`,如果帳號不對記得改。

## 之後怎麼用在 104 履歷上
等圖片和文字都確認好,建議把這個資料夾放到 GitHub Pages 或 Netlify(拖檔案上傳即可,免費)
產生一個網址,貼到 104 履歷的自我介紹欄位裡,變成一個可以直接點開的連結,
比附件檔案更容易被 HR 看到。
